AI at Domo: Getting Started

A practical guide to using Claude at Domo. Complete the checklist below, then use the reference sections to go deeper.

Checklist

Getting Started Checklist

Work through these in order. Each one builds on the last. Reference sections below explain the concepts in more detail.

1
Download Claude and introduce yourself
  1. Go to claude.ai and sign up with your work email, or download the Desktop app at claude.ai/download.
  2. Go to your account Settings and find Custom Instructions (or User Profile). Tell Claude your name, role, team, what you work on, and how you like to communicate.
  3. Start a conversation and ask:
    What else would help you know about me to work more effectively together?
  4. Ask Claude to save key context to memory:
    Save these to memory: [your key priorities or preferences]
2
Connect to Domo (Domo MCP)

First, get a Domo Developer Token

  1. Log into domo.domo.com, click your name, and go to Admin.
  2. Go to More → Developer → Access Tokens.
  3. Click Generate Access Token, name it Claude MCP, and copy the token.

In Claude Cowork

  1. Open the Developer panel and click Edit Config.
  2. Copy everything you see and paste it into a Claude conversation.
  3. Ask Claude to add the Domo MCP entry and return the complete updated config with a placeholder for your token. Do not share your token in the conversation.
  4. Paste the result back, replace the placeholder with your actual token, and save.

In Claude Code

  1. Ask Claude:
    Help me set up Claude Code on my [Mac / Windows PC]. Walk me through the full installation.
  2. Once running, create a .mcp.json file in your workspace folder with the config below.
  3. Replace YOUR_TOKEN_HERE with your actual token.

Config (for either Cowork or Claude Code):

.mcp.json 
{
  "mcpServers": {
    "domo-mcp": {
      "type": "http",
      "url": "https://domo.domo.com/api/ai/v1/assistant/mcp/toolkits/DOMO_BASIC_MCP",
      "headers": {
        "X-DOMO-DEVELOPER-TOKEN": "YOUR_TOKEN_HERE"
      }
    }
  }
}
Test it: Ask Claude to "Search for a revenue dataset in Domo." If it returns results, you're connected. Then try: "Show me my top customers by ARR this quarter."
3
Build a Project
  1. Find a document you want to be able to ask questions about: a contract, an agreement, a policy, a report.
  2. In Claude Chat, go to Projects and create a new one.
  3. Name it after the subject (e.g., "Q1 Debt Agreement Review").
  4. Upload the document and add a brief instruction:
    You are reviewing [document name] from the perspective of [your role]. Help me understand key terms, flag anything notable, and answer questions about what it means for Domo.
  5. Ask your first question:
    Summarize the key terms and flag anything I should pay attention to.
4
Build a Skill
  1. Identify a task you do regularly that takes time but follows a consistent pattern. Say:
    I want to build a Skill, an automated workflow I can trigger with a command. Here are tasks I do regularly: [list 3–5 things]. Help me pick the best candidate and sketch out what it should do.
  2. Once you've chosen, ask Claude to draft it:
    Draft a SKILL.md file for [the task]. It should trigger when I say /[skill-name] and follow these steps: [what you described].
  3. Save the file to skills/[skill-name]/SKILL.md in your workspace.
  4. Trigger it and adjust the instructions until it runs the way you want.
  5. Remember: if Claude does something useful in any conversation, say "Make everything we've just done into a Skill" and it will package it up for you.
Bonus

Want to go further? Ask Claude: "What can I build with Claude Code and the Domo MCP?" You can build native Domo apps, custom dashboards, and Procode apps using React and Domo's developer framework.

Key concepts: Projects, Connectors, and Skills

What is a Project?

A Project is a scoped workspace where you give Claude deep context on a specific subject, usually by uploading the relevant documents. Every conversation inside that Project starts with Claude already knowing those materials.

Use a Project when you have a document you want to ask questions about repeatedly: a contract, agreement, policy, or report. You get precise, contextual answers without pasting the document each time.

What is a Connector?

A Connector (sometimes called an MCP Server) links Claude to an external system: Domo, Outlook, Gmail, SharePoint, Ramp, and others. Once connected, Claude can query live data, read emails, pull from calendar, and search documents without any manual copy-paste.

Use a Connector when you want Claude to pull live data or interact with a system on your behalf. With the Domo Connector, you can ask "What are my top 10 customers by ARR this quarter?" and Claude queries Domo directly.

What is a Skill?

A Skill is a saved set of instructions that Claude follows when you trigger it. Think of it like a custom command. You say /my-skill (or a phrase you define), and Claude executes a specific workflow you've designed.

Use a Skill when you do the same type of task on a recurring basis and want consistent, repeatable output every time. Skills can also be shared with your team.

Tip: You don't have to design Skills from scratch. If Claude does something useful in any conversation, say "Make everything we've just done into a Skill" and it will package the workflow for you.
Setting yourself up: Custom Instructions, CLAUDE.md, and Memory

Custom Instructions (Chat and Desktop App)

In Claude Chat and the Desktop app, go to your account Settings and look for Custom Instructions (also called User Profile or Preferences). This is where you describe yourself once and it applies to every conversation.

Tell Claude your name, role, team, what you work on, and how you like to communicate. Think of it as an onboarding document for a new colleague.

CLAUDE.md (Claude Code only)

If you're using Claude Code, create a file called CLAUDE.md in your workspace. Claude reads it at the start of every Code session. You don't write it manually — open Claude Code and say "I want to create a CLAUDE.md file. Ask me questions about who I am, what I do, and how I like to work, then draft the file."

Memory

Memory lets Claude remember things across conversations. Unlike your Custom Instructions (a standing introduction), Memory builds up over time as you work.

Update it by telling Claude "Remember that..." and it will save it. Ask "What do you have in memory about me?" at any time to see what's been saved.

Data and privacy

Domo uses Claude through a Teams plan. Your conversations are private. The Teams plan provides enterprise-level privacy: conversations are not used to train Claude's models and are not shared with Anthropic beyond what is needed to run the service.

That said, use judgment. Some categories of information should stay out of any AI tool without specific confirmation of the right setup:

  • Material non-public information (un-issued financials, M&A activity, earnings before filing)
  • Customer PII or anything covered by a customer NDA
  • Personally identifiable employee information
  • Anything where AI assistance could affect a compliance or audit finding
Practical test: If you would hesitate before pasting something into an external email, apply the same pause before putting it in Claude. When in doubt, ask before you proceed.
Building your first Skill

The most common question when getting started: what should I automate?

Start here: What is something you do on a daily, weekly, or monthly basis that takes some time and you do fairly regularly? That is your first Skill candidate.

Before you start building, write down:

  1. What is the task?
  2. What do you start with? (an email, a dataset, a document, a meeting?)
  3. What does the output look like? (a summary, a table, a draft, a set of action items?)
  4. How often do you do it?

Then share it with Claude and ask it to help you turn it into a Skill.

Types of Skills worth building

  • Recurring reports — something you pull and format on a schedule
  • Inbox triage — scan flagged emails, summarize what needs attention, list action items
  • Morning briefing — calendar, open tasks, and priorities in one focused plan
  • Document review — check a document against a standard, flag issues, suggest edits
  • Data pulls and analysis — query a dataset, filter it, summarize the key takeaways
  • Meeting prep — pull notes, tasks, and context before a recurring meeting
Tips for working with Claude

Ask Claude how to use Claude. If you're unsure how to do something, just ask. Describe what you're trying to do and Claude will walk you through it. This is the fastest way to learn.

Start with Claude Chat. Move to Cowork when you're ready to do file-based work or run Skills. Come to Claude Code when you want to build something or connect to Domo data directly.

Start with context, every time. The more Claude knows about you upfront, the better every response will be. Your Custom Instructions and Memory investment pays off immediately.

Be specific. Instead of "summarize this," try "summarize the key risks in 3 bullet points for a CFO audience." Claude responds to specificity.

Iterate, don't restart. Claude holds context inside a conversation. If the first output isn't right, tell it exactly what to change. You don't need to re-explain everything.

When in doubt, describe what you want. You don't need to know the right syntax or command. Just describe your goal in plain language and Claude will figure out how to get there.

Turn good work into a Skill. If Claude does something useful in any conversation, say "Make everything we've just done into a Skill." It will package the workflow into a reusable command you can trigger anytime.

Use the same judgment you would with email. If you would hesitate before pasting something into an external email, apply the same pause before putting it in Claude.
